C. IMAGE STORAGE STRATEGY
=========================

For now, keep the current simple strategy.

If the user uploads a file:

* Convert it to a base64 data URL in the frontend.
* Store it in the existing image field:

  * News: imageUrl
  * Articles: imageUrl or coverImageUrl according to current schema mapping

If the user uses a URL:

* Store the URL string in the same image field.

Do not add:

* Firebase Storage
* S3
* external upload services
* new file server

Important:
The frontend image display must handle both:

* normal https image URLs
* data:image/...;base64 images

==================================================
D. SERVER / BACKEND REQUIREMENTS
================================

Backend must accept both:

* https/http image URL
* base64 data image URL

For base64:
Validate:

* starts with data:image/
* mime is jpeg/png/webp
* decoded size <= 2 MB

For URL:
Validate lightly:

* must be a string
* if provided, should be a valid http/https URL
* or allow empty/null

Do not allow dangerous non-image protocols:

* javascript:
* data:text/html
* file:
* blob: should not be persisted

If invalid:
Return 400 with Arabic-friendly error.
