BOARD MEMBERS BOARD-TYPE TRANSFER FIX — SGMA APP

Start this task now.

Scope:
Improve Board of Directors management so each board member card can be moved between board sections instead of deleting/recreating cards.

Current problem:
When a new board cycle starts, admins currently would need to delete/recreate board member cards. This is wrong. Each board member card should have a selectable board category/type, and changing it should immediately move the card to the selected page/section.

Required:
Add or improve a board type selector for each board member:

1. مجلس الإدارة الحالي
2. مجلس الإدارة السابق
3. أرشيف مجالس الإدارة

Do not rebuild the board module from scratch.
Do not change unrelated modules.
Do not break current board management, images, bio bullet formatting, or permissions.

Current architecture:

* PostgreSQL + Drizzle
* JWT auth
* Arabic RTL
* Mobile-first
* No Firebase
* No MySQL migration

==================================================
A. REQUIRED BEHAVIOR
====================

Each board member record must have a boardType/category.

Allowed boardType values:

* CURRENT
  Label: مجلس الإدارة الحالي

* PREVIOUS
  Label: مجلس الإدارة السابق

* ARCHIVE
  Label: أرشيف مجالس الإدارة

Behavior:

* If admin changes boardType from CURRENT to PREVIOUS, the card disappears from current board page and appears in previous board page.
* If admin changes boardType to ARCHIVE, it appears in archive page.
* No deletion/recreation needed.
* The member card data stays the same:

  * name
  * position
  * bio
  * phone
  * email
  * image
  * displayOrder
  * active status
