VOLUNTEER DELEGATIONS REGISTRATION MODULE — SGMA APP

Start this task now.

Scope:
Add a new page under /more for members to register for volunteer delegations.

Page name:
تسجيل للوفود

Purpose:
Any authenticated member can submit volunteer delegation information.
MODERATOR / ADMIN / SUPER_ADMIN can view submitted registrations as cards, including uploaded PDF files.

Do not add unrelated features.
Do not break existing modules:

* login
* register
* home/my account
* news
* articles
* chat
* broadcast
* admin dashboard
* board
* more page
* developer-info
* static pages
* membership approval flow
* branding/logo

Current architecture:

* PostgreSQL + Drizzle
* JWT auth
* Arabic RTL
* Mobile-first
* No Firebase
* No MySQL migration
* No old SGMA APP1 routes

C. MEMBER FORM FIELDS
=====================

Create a clean Arabic RTL mobile-first form.

Fields:

1. الاسم الكامل عربي
   field:
   fullNameArabic
   required

2. الاسم الكامل ألماني
   field:
   fullNameGerman
   required

3. رقم الهاتف
   field:
   phone
   required

4. رقم الواتساب
   field:
   whatsapp
   required

5. عنوان الإيميل
   field:
   email
   required, valid email

6. تاريخ النزول إلى سوريا من
   field:
   travelToSyriaFrom
   required, date

7. تاريخ النزول إلى سوريا إلى
   field:
   travelToSyriaTo
   required, date

8. تاريخ العمل التطوعي
   field:
   volunteerWorkDateText
   required, free text

Label:
تاريخ العمل التطوعي من / إلى

Placeholder:
مثال: من 01.08.2026 إلى 15.08.2026 أو حسب التنسيق المتفق عليه

9. نوع العمل التطوعي
   field:
   volunteerWorkType
   required, free text

Placeholder:
مثال: محاضرات، ورشات عمل، عمليات جراحية، تدريب كوادر، دعم لوجستي...

10. الاختصاص
    field:
    professionGroup
    required, dropdown

Use the same options as registration:

* Arzt / طبيب
* Pflege / تمريض
* ATA
* OTA
* Physiotherapie / علاج فيزيائي
* Labor / مخبر
* Radiologie / أشعة
* Medizintechnik / هندسة طبية
* Rettungsdienst / إسعاف
* Student / طالب
* Andere / أخرى

11. التخصص مع الشهادة
    field:
    specialtyWithCertificate
    required, free text

Placeholder:
مثال: Fachweiterbildung Anästhesie und Intensivpflege, Chirurgie, Innere Medizin, Pflegefachkraft...

12. هل تحتاج مساعدة في ترخيص مزاولة المهنة في سوريا؟
    field:
    needsSyrianPracticeLicenseHelp
    required boolean yes/no

Options:

* نعم
* لا

13. هل لديك معدات لوجستية تأخذها معك أثناء السفر؟
    field:
    hasLogisticsEquipment
    required boolean yes/no

Options:

* نعم
* لا

If yes:
Show required free-text field:
equipmentDetails

Label:
اذكر المعدات

Placeholder:
اكتب نوع المعدات، الكمية، وهل تحتاج إلى وثائق أو تنسيق خاص...

If no:
equipmentDetails optional/empty.

14. رفع ملفات PDF
    field:
    attachments

Label:
إرفاق ملفات PDF

Description:
يمكنك رفع شهادات، فواتير معدات، أو وثائق داعمة بصيغة PDF.

Requirements:

* PDF files only
* Allow multiple PDFs
* Max file size: 5 MB per file
* Max files: 5
* Show selected file names
* Allow removing selected files before submit

Arabic errors:

* يسمح برفع ملفات PDF فقط
* حجم الملف كبير جداً، الحد الأقصى 5MB
* لا يمكن رفع أكثر من 5 ملفات

D. SAVE AS PDF BUTTON
=====================

Add button:
حفظ المعلومات كملف PDF

Behavior:

* Generate a client-side PDF containing the filled form information.
* The member can download/save it before or after submission.
* PDF should be Arabic RTL as much as possible.
* Include title:
  استمارة تسجيل للوفود التطوعية
* Include date of generation.
* Include all form fields.
* Do not include uploaded PDF file contents inside the generated PDF; list their filenames only.

Important:
This button is for the member to keep a personal copy.
It must not submit the form.

If full Arabic PDF generation is technically difficult, generate a clean printable HTML view and call browser print/save as PDF, but prefer direct PDF download if stable.

Button text:
حفظ المعلومات كملف PDF

Fields:

* requestId
* fileName
* mimeType
* fileSize
* fileData
* createdAt

Do not add:

* Firebase Storage
* S3
* external file services

Security:

* Only PDF files accepted.
* Max 5MB each.
* Max 5 files per request.

G. BACKEND API REQUIREMENTS
===========================

Create APIs:

Member submit:
POST /api/volunteer-delegations

* JWT protected
* ACTIVE user only
* validates required fields
* validates files
* userId comes from JWT
* never trust frontend userId
* creates request and attached files

Member own requests:
GET /api/volunteer-delegations/my

* JWT protected
* returns current user’s submitted requests
* include attachment metadata, not full file data by default

Admin/moderator list:
GET /api/admin/volunteer-delegations

* MODERATOR / ADMIN / SUPER_ADMIN only
* returns submitted requests as cards
* include user info if useful
* include attachment metadata

Admin/moderator detail:
GET /api/admin/volunteer-delegations/:id

* MODERATOR / ADMIN / SUPER_ADMIN only
* returns full details

Download file:
GET /api/admin/volunteer-delegations/files/:fileId

* MODERATOR / ADMIN / SUPER_ADMIN only
* returns/downloads the PDF file

Optional member file access:
GET /api/volunteer-delegations/files/:fileId

* only owner can download their own file

Update status:
PATCH /api/admin/volunteer-delegations/:id

* MODERATOR / ADMIN / SUPER_ADMIN only
* update:

  * status
  * adminNotes

Allowed statuses:

* SUBMITTED
* IN_REVIEW
* ACCEPTED
* REJECTED
* ARCHIVED
