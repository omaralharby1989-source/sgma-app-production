I. BACKEND API REQUIREMENTS
===========================

Create clean JWT-protected APIs.

Public/member read:
GET /api/board/members
Query:

* boardType optional, default CURRENT

Returns active members only, ordered by displayOrder then createdAt.

GET /api/board/members/:id
Returns one active member if needed.

Admin/Super management:
POST /api/board/members

* ADMIN / SUPER_ADMIN only
* creates board member
* validates fields
* image optional
* server sets createdById from JWT if field exists

PATCH /api/board/members/:id

* ADMIN / SUPER_ADMIN only
* updates board member
* server sets updatedById from JWT if field exists

DELETE /api/board/members/:id

* ADMIN / SUPER_ADMIN only
* soft delete:
  isActive = false

Security:

* MEMBER cannot create/edit/delete
* MODERATOR cannot create/edit/delete
* Backend must return 403 for unauthorized actions
* Do not rely only on frontend hiding

Status codes:

* 400 invalid input
* 401 unauthenticated
* 403 unauthorized
* 404 not found
