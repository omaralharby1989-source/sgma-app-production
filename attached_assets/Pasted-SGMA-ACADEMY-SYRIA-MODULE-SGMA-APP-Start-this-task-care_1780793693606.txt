B. USER ACCESS MODEL
====================

Add a new user access field:

accessScope

Allowed values:

* FULL_APP
* SYRIA_ACADEMY_ONLY

Default for existing users:
FULL_APP

Meaning:

1. FULL_APP
   Regular SGMA member.
   Can use the normal app according to existing role permissions.
   Can also access SGMA Academy.
   Can view ALL academy lectures across ALL specialties.

2. SYRIA_ACADEMY_ONLY
   Syria Academy-only user.
   Can access only:

* Academy pages
* public news list/detail
* published articles list/detail in read-only mode
* My Account
* legal/static pages
* developer info if currently public/allowed

Cannot access:

* write article
* my articles
* chat
* admin direct chat
* volunteer delegations
* board of directors
* task management
* admin dashboard
* broadcast management
* ads management
* any full-member-only feature

Backend and frontend must both enforce this.
Do not rely only on hidden links.

C. MEMBERSHIP NUMBER RULES
==========================

Current rule for SGMA Germany/full members:
Membership number is numeric only.

Valid examples:

* 1
* 25
* 798

Invalid for full member registration:

* SGMA123
* DE123
* random text

New rule for Syria Academy registration:
The user may enter exactly:

SY

in the membershipNumber field.

If membershipNumber is exactly SY:

* Treat the registration as a Syria Academy application.
* Create the user as PENDING / inactive, same as current approval flow.
* Set accessScope = SYRIA_ACADEMY_ONLY.
* Store a temporary/pending membership marker as SY or equivalent until admin activation.
* Do NOT issue token after signup.
* Admin must activate the account.

On admin activation:

* Generate the final Syria academy membership number automatically:
  SY1, SY2, SY3, ...
* Use the next available number.
* Save the final value in membershipNumber.
* Activate the account:
  status = ACTIVE
  isActive = true

Important:
Do not let normal users choose SY1/SY2 manually during registration.
At registration time, Syria users enter only SY.

Registration helper text under membershipNumber:
لأعضاء SGMA: أدخل رقم عضويتك فقط، مثال: 798.
لأعضاء أكاديمية سوريا: أدخل SY فقط.

E. EXTRA ACCESS OVERRIDES FOR SYRIA USERS
=========================================

Admin should be able to grant additional academy specialty access to Syria users.

Add optional field/model:
academyAllowedSpecialties

For SYRIA_ACADEMY_ONLY user:
Default lecture access:

* GENERAL lectures
* user.academySpecialty lectures

Additional access:

* any specialties listed in academyAllowedSpecialties

Admin option:

* Grant one or more extra specialties.
* Grant all specialties if needed.

Suggested special value:
ALL

Rules:

* If academyAllowedSpecialties includes ALL, the Syria user can view all academy lectures.
* If not, they can view only GENERAL + own academySpecialty + explicitly allowed specialties.

FULL_APP users:
Always view all academy lectures regardless of academySpecialty.

* Pending/inactive behavior remains unchanged.

* After login, SYRIA_ACADEMY_ONLY users should be redirected to /academy, not /home.

* createdAt
* updatedAt

* DRAFT
* PUBLISHED
* HIDDEN
* ARCHIVED
