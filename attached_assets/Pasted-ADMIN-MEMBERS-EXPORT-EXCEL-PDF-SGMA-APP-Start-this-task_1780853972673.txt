ADMIN MEMBERS EXPORT — EXCEL / PDF — SGMA APP

Start this task now.

Scope:
Add export functionality to the existing Admin Users / Member Management page.

Goal:
ADMIN and SUPER_ADMIN can export member data to Excel or PDF:

* all members
* selected members
* filtered members by profession/specialty/access scope/role/status

The exported file must include full member information and relevant activity summaries.

Do not add unrelated features.
Do not rebuild Admin Users from scratch.
Do not change existing permissions except adding this export feature.
Do not break current stable modules.

Existing stable modules must remain working:

* login

* visual identity

Current architecture:

* PostgreSQL + Drizzle

* No MySQL migration

B. PERMISSIONS
==============

Allowed:

* ADMIN
* SUPER_ADMIN

Not allowed:

* MODERATOR
* MEMBER
* SYRIA_ACADEMY_ONLY users

Rules:

1. ADMIN can export:

   * MEMBER
   * MODERATOR
   * Syria Academy users
   * filtered groups
   * selected users
   * all non-SUPER_ADMIN users if this matches current admin visibility

2. ADMIN must NOT export SUPER_ADMIN full private data unless existing policy already allows it.
   Recommended:

   * ADMIN export excludes SUPER_ADMIN users.

3. SUPER_ADMIN can export all users, including ADMIN / MODERATOR / MEMBER / Syria Academy users.

Backend enforcement mandatory.
Frontend hiding alone is not enough.

D. EXPORT DATA CONTENT
======================

The export file should include complete member profile information and activity summaries.

Member profile fields:

* id
* fullName
* account
* email
* membershipNumber
* role
* status
* isActive
* isDeveloper
* accessScope
* academySpecialty
* academyAllowedSpecialties
* phone
* whatsapp
* birthDate
* address
* professionGroup
* specialtyText
* bio
* createdAt
* updatedAt

Activity summary fields:

* articlesCount
* approvedArticlesCount
* pendingArticlesCount
* newsCreatedCount if relevant
* tasksAssignedCount
* taskReportsCount
* volunteerDelegationRequestsCount
* academyLecturesViewedCount if views table exists
* academyAccessSummary
* lastActivityAt if computable

If some activity is not available or no table exists:

* include 0 or "غير متاح"
* document this clearly in report.

Do not export:

* passwordHash
* JWT tokens
* secrets
* DATABASE_URL
* private system config

Privacy:

* For chat, do NOT export full chat message contents by default.
* If including chat, include only counts:

  * publicChatMessagesCount
  * adminDirectMessagesCount
    Not full messages.

G. BACKEND API REQUIREMENTS
===========================

Create backend endpoints:

GET /api/admin/users/export
Query params:

* format: xlsx | csv | pdf | json
* role optional
* status optional
* accessScope optional
* professionGroup optional
* academySpecialty optional
* search optional
* selectedIds optional comma-separated
* includeActivity optional boolean default true

Access:

* ADMIN
* SUPER_ADMIN

Rules:

* ADMIN cannot export SUPER_ADMIN users.
* SUPER_ADMIN can export all.
* Server must apply permission filters regardless of client request.
* Never expose passwordHash or secrets.

Response:

* For xlsx/csv/pdf: downloadable file.
* For json: preview data if useful.

If implementing file generation on frontend is easier:

* Backend may return JSON export data securely.
* Frontend generates Excel/PDF.
* But backend must still enforce data visibility and remove sensitive fields.

Preferred:
Backend returns sanitized export data; frontend generates file if already using client export patterns.
Choose safest and fastest implementation.
