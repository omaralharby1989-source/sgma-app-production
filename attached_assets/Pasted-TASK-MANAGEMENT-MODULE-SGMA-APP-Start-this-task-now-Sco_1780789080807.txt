TASK MANAGEMENT MODULE — SGMA APP

Start this task now.

Scope:
Build a complete internal Task Management module for SGMA APP.

Purpose:
Admins, moderators, and super admins can create and assign tasks to specific members.
Assigned members can access only their own tasks and submit detailed reports with attachments from their phone/device.
Admins/moderators/super admins can view all tasks, all reports, all attachments, and export task information to Excel or PDF for archiving.

Do not add unrelated features.
Do not break existing stable modules:

* login
* register
* home/my account
* news
* articles
* chat
* broadcast
* admin dashboard
* board
* volunteer delegations
* ads management
* more page
* developer-info
* static pages
* membership approval flow
* logo/branding

Current architecture:

* PostgreSQL + Drizzle
* JWT auth
* Arabic RTL
* Mobile-first
* No Firebase
* No MySQL migration
* No old SGMA APP1 routes

==================================================
A. MAIN IDEA
============

Create a Trello-like task system, but without complex drag/drop for now.

Use clean task cards, filters, details page, reports, attachments, and export buttons.

Roles:

1. SUPER_ADMIN
   Full task control:

* create tasks
* assign tasks to one or multiple members
* view all tasks
* view all reports
* add admin reports/comments
* change status
* export Excel/PDF
* download attachments

2. ADMIN
   Same full task control:

* create tasks
* assign tasks
* view all
* reports
* status changes
* export Excel/PDF
* download attachments

3. MODERATOR
   Same full task control:

* create tasks
* assign tasks
* view all
* reports
* status changes
* export Excel/PDF
* download attachments

4. MEMBER
   Limited:

* can access task module ONLY if assigned to at least one task
* can see only own assigned tasks
* can submit detailed reports
* can upload attachments from phone/device inside reports
* cannot see other members’ tasks
* cannot create tasks
* cannot export all tasks
* cannot access admin task list

Important:
A normal MEMBER with no assigned tasks must NOT see task links and must NOT access task pages.

Backend enforcement is mandatory.

* id

* MEDIUM
* HIGH
* URGENT

Status values:

* NEW
* IN_PROGRESS

* COMPLETED
* POSTPONED
* CANCELLED

priority = MEDIUM

2. task_assignees

Fields:

* id
* taskId
* userId
* assignedById
* assignedAt
* isActive boolean default true

Rules:

* A task can have one or multiple assigned members.
* A member can have many tasks.
* Use this table for visibility control.

3. task_reports

Fields:

* id
* taskId
* authorId
* reportText
* progressPercent nullable integer
* reportType
* createdAt
* updatedAt

reportType values:

* MEMBER_REPORT
* ADMIN_NOTE

Rules:

* Assigned member can create MEMBER_REPORT for own task.
* MODERATOR / ADMIN / SUPER_ADMIN can create ADMIN_NOTE on any task.
* Staff can also add a report if needed.

4. task_report_attachments

Fields:

* id
* reportId
* fileName
* mimeType
* fileSize
* fileData
* createdAt

Storage strategy:
For now, store attachments as base64 data URLs or base64 text in DB.

Allowed attachment types:

* PDF
* JPG/JPEG
* PNG
* WEBP
* DOC/DOCX if easy and safe
* XLS/XLSX if easy and safe

Recommended minimum:

* PDF + images only

Max files:

* 5 files per report

Max file size:

* 5 MB per file

Do not add:

* Firebase Storage
* S3
* external file service

M. BACKEND API REQUIREMENTS
===========================

Create APIs:

Member/staff read own/admin tasks:

GET /api/tasks/my

* Authenticated active user
* Returns tasks assigned to current user
* Staff may also use admin endpoint for all tasks

GET /api/tasks/:id

* Member can access only if assigned
* Staff can access any task

Create member report:
POST /api/tasks/:id/reports

* Member assigned to task can submit MEMBER_REPORT
* Staff can submit ADMIN_NOTE or staff report
* Attachments supported

Download attachment:
GET /api/tasks/reports/attachments/:attachmentId

* Member can download only if attachment belongs to a task assigned to them
* Staff can download any task attachment

Admin/staff endpoints:

GET /api/admin/tasks

* MODERATOR / ADMIN / SUPER_ADMIN only
* list all tasks with filters

POST /api/admin/tasks

* MODERATOR / ADMIN / SUPER_ADMIN only
* create task and assignees

PATCH /api/admin/tasks/:id

* MODERATOR / ADMIN / SUPER_ADMIN only
* update task fields/status/assignees/adminNotes as needed

GET /api/admin/tasks/export

* MODERATOR / ADMIN / SUPER_ADMIN only
* returns data for export or generates CSV/PDF depending design

Optional:
GET /api/admin/tasks/:id

* staff detail if separate from /api/tasks/:id

Security:

* userId comes from JWT
* assignedById/createdById from JWT
* members cannot access unassigned tasks
* members cannot create tasks
* members cannot update task status directly
* staff can see all
