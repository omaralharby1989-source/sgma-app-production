G. BACKEND API REQUIREMENTS
===========================

Create clean JWT-protected chat APIs.

Public chat endpoints:

GET /api/chat/public/messages

* Returns latest public messages.
* Must require JWT.
* Should limit results reasonably, for example latest 100 messages.

POST /api/chat/public/messages

* Creates message for authenticated user.
* Requires content.
* Trims content.
* Rejects empty messages.

PATCH /api/chat/public/messages/:id

* Allows sender to edit own message.
* Rejects editing deleted messages.
* Does not allow editing another user’s message.

DELETE /api/chat/public/messages/:id

* Soft delete.
* Sender can delete own message.
* MODERATOR / ADMIN / SUPER_ADMIN can delete any message.

Admin direct chat endpoints:

GET /api/chat/admin/conversations

* For MODERATOR / ADMIN / SUPER_ADMIN only.
* Returns member conversation list.
* Must not be available to MEMBER.

GET /api/chat/admin/messages

* For MEMBER: returns own admin conversation.
* For admin roles: can use userId query to return selected user conversation.

POST /api/chat/admin/messages

* MEMBER sends to administration.
* Admin roles can reply to selected member.
* Validate receiver logic carefully.

PATCH /api/chat/admin/messages/:id

* Sender can edit own message only.
* Deleted messages cannot be edited.

DELETE /api/chat/admin/messages/:id

* Sender can delete own message.
* MODERATOR / ADMIN / SUPER_ADMIN can delete any direct chat message.

Security:

* Every endpoint must use authenticated user from JWT.
* Never trust senderId from the frontend.
* Never allow MEMBER to read another member’s direct chat.
* Never allow MEMBER to impersonate admin.
* Never allow role escalation.
