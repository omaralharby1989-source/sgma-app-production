C. CAPACITOR SETUP
==================

If Capacitor is not installed, add and configure it properly.

Required:

* @capacitor/core
* @capacitor/cli
* @capacitor/android

Create/update:
capacitor.config.ts

Required config:

* appId: org.sgma.app
* appName: SGMA APP
* webDir: correct Vite build output, usually dist/public or dist depending on project
* bundledWebRuntime: false if applicable

Make sure the build uses the actual frontend production build.

D. ANDROID PROJECT
==================

Add Android platform if not already present:

npx cap add android

If Android already exists:

* do not recreate destructively
* inspect and update safely

Then run:

* frontend build
* npx cap sync android

Do not run destructive commands.
Do not reset the repository.
Do not delete existing app files.
Do not change backend logic.

==================================================
E. PRODUCTION BACKEND / API URL CHECK
=====================================

Critical:
The mobile app must not be hardcoded to a temporary localhost URL.

Check how the frontend calls API routes.

If the app uses relative API paths like /api/...:

* verify whether this works inside Capacitor.
* If not, propose a safe configuration using a production backend base URL.

Do NOT invent or hardcode a production domain unless provided.
If a production backend URL is missing, report this as a blocker.

Expected report wording:
“Production backend URL is required before final store submission.”

If the app is still using Replit preview URL:

* report this clearly.
* do not silently lock the app to an unstable preview URL unless explicitly approved.
