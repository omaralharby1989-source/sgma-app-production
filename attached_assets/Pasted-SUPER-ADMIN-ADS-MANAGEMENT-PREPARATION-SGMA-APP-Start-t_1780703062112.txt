SUPER ADMIN ADS MANAGEMENT PREPARATION — SGMA APP

Start this task now.

Scope:
Prepare the Ads Management system so only SUPER_ADMIN can control advertisements.

Important:
Do NOT connect Google AdSense yet.
Do NOT add real Google AdSense script now.
Do NOT ask for AdSense client id now.

This phase is only to prepare:

1. Ads management page for SUPER_ADMIN only.
2. Internal custom ads system.
3. Google AdSense settings placeholder/config system for future activation.
4. Global ad display component at the bottom of selected pages.
5. Ability for SUPER_ADMIN to enable/disable ads and control where they appear.

Do not add unrelated features.
Do not break existing modules:

* login
* register
* home/my account
* news
* articles
* chat
* broadcast
* admin dashboard
* board
* volunteer delegations
* more page

* branding/logo

Current architecture:

* PostgreSQL + Drizzle
* JWT auth
* Arabic RTL
* Mobile-first
* No Firebase
* No MySQL migration
* No old SGMA APP1 routes

D. ADS SETTINGS MODEL
=====================

Create a database table for global ads settings.

Suggested table:
ad_settings

Fields:

* id
* adsEnabled boolean default false
* googleAdsEnabled boolean default false
* googlePublisherId nullable text
* googleAdSlotBottom nullable text
* showOnHome boolean default false
* showOnNews boolean default true
* showOnArticles boolean default true
* showOnBoard boolean default true
* showOnMore boolean default true
* showOnStaticPages boolean default true
* showOnChat boolean default false
* showOnAdmin boolean default false
* showOnAuthPages boolean default false
* updatedById nullable
* updatedAt

Rules:

* There should be only one settings row.
* Self-seed default row if missing.
* Default adsEnabled = false.
* Default googleAdsEnabled = false.
* showOnAuthPages must remain false by default.

Important:
Even if googleAdsEnabled is true later, Google script should NOT be loaded until we explicitly add the real AdSense client ID phase.

I. BACKEND API REQUIREMENTS
===========================

Create APIs:

Read public/app ad state:
GET /api/ads/active

* JWT protected
* Returns:

  * settings relevant for display
  * active custom ads for current placement if query is provided
* Normal users can read active ads only.
* Do not expose sensitive config unnecessarily.

Query:
placement optional

Super admin management:
GET /api/admin/ads/settings

* SUPER_ADMIN only

PATCH /api/admin/ads/settings

* SUPER_ADMIN only

GET /api/admin/ads/custom

* SUPER_ADMIN only

POST /api/admin/ads/custom

* SUPER_ADMIN only

PATCH /api/admin/ads/custom/:id

* SUPER_ADMIN only

DELETE /api/admin/ads/custom/:id

* SUPER_ADMIN only
* Prefer soft deactivate/archive instead of hard delete if current style prefers that.

Security:

* ADMIN must receive 403
* MODERATOR must receive 403
* MEMBER must receive 403
* Unauthenticated gets 401
