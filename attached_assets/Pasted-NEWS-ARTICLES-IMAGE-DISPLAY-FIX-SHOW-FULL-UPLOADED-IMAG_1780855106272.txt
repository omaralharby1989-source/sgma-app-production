Required:
Images must display fully without cropping.

Do not change upload logic.
Do not change database schema.
Do not change backend validation.

==================================================
A. IMAGE DISPLAY REQUIREMENT
============================

For News and Articles images:

1. Do not crop uploaded images.
2. Do not use object-cover where it hides parts of the image.
3. Use object-contain or adaptive image height.
4. The card/image container should adapt to the uploaded image ratio as much as possible.
5. The full image must be visible.
6. No broken layout on mobile.
7. No horizontal overflow.

Affected areas:

* /news list cards
* /news/:id detail image
* /articles list cards
* /articles/:id detail image
* any admin preview cards if present

==================================================
B. CARD IMAGE BEHAVIOR
======================

On list cards:

Preferred behavior:

* Show the full image inside a clean container.
* Use:
  object-contain
  width: 100%
  height: auto
  max-height reasonable for mobile

Suggested:

* image wrapper background: soft blue/white or muted background
* border radius consistent with card
* image class:
  w-full h-auto object-contain
* if a fixed wrapper is needed:
  use max-h-[280px] or similar and object-contain, not object-cover

Important:
Do not force all images into the same cropped 16:9 box if it cuts important content.

If the uploaded image is vertical:

* card should show the full vertical image
* it may be taller, but still clean

If the uploaded image is horizontal:

* show full horizontal image

==================================================
C. DETAIL PAGE IMAGE BEHAVIOR
=============================

On news/article detail pages:

* Display the image full width.
* Preserve aspect ratio.

* Use:
  w-full h-auto object-contain
* Optional max-height:
  max-h-[70vh]
  so very tall images do not dominate the whole screen.
* Use a soft background behind transparent/contained images if needed.
