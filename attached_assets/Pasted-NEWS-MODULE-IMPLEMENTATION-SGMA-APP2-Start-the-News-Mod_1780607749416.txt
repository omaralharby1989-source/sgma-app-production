==================================================
A. NEWS MODULE PURPOSE
======================

The News Module allows users to read SGMA news in a clean, visual, mobile-first interface.

Users should see news as beautiful image-based cards.

When the user taps a news card, the full news article opens on a dedicated detail page.

The design should feel like a professional association/news app, not a basic admin table.

==================================================
B. REQUIRED ROUTES
==================

Create and activate these routes:

1. /news
   News list page.

2. /news/:id
   News detail page.

Both routes must be JWT protected for now.

Unauthenticated users must be redirected to /login.

Update bottom navigation:

* الأخبار must open /news
* المحادثات must still open /chat
* حسابي must still open /home
* المزيد must still open /more

Do not create broken nav links.

F. BACKEND API REQUIREMENTS
===========================

Create clean JWT-protected APIs.

Required endpoints:

GET /api/news
Returns list of published news only.
Sorted newest first.

GET /api/news/:id
Returns one published news item by id.

For testing and seed only:
If there is no admin UI yet, create seed/sample news data safely.

Do not create public write endpoints for normal members.

Important:
Members must NOT be able to create/edit/delete news in this phase.

Optional but acceptable if useful:
Create staff-only backend endpoints for future admin use, but do not build full admin UI now:

* POST /api/news
* PATCH /api/news/:id
* DELETE /api/news/:id

If you add these, they must be protected:
Only MODERATOR / ADMIN / SUPER_ADMIN can create/edit/archive news.
MEMBER cannot.

But frontend creation UI is NOT required now unless explicitly asked later.
