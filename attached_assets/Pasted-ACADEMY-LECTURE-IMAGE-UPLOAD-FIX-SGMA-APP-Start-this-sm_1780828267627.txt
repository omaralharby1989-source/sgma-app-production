ACADEMY LECTURE IMAGE UPLOAD FIX — SGMA APP

Start this small fix now.

Scope:
Allow academy lecture announcements and recordings to use an image uploaded from phone/device, not only an image URL.

Current problem:
In SGMA Academy, when creating/editing a lecture announcement or recorded lecture, the thumbnail/image field should support uploading an image from the device.

Required:
In /admin/academy create/edit lecture form, the image/thumbnail field must support both:

1. Image URL
2. Upload image from phone/device

Do not rebuild the Academy module.
Do not change unrelated modules.
Do not change academy permissions.
Do not change video logic.
Do not change Google Drive embed logic.
Do not change accessScope/SY membership logic.

Current architecture:

* PostgreSQL + Drizzle
* JWT auth
* Arabic RTL
* Mobile-first
* No Firebase
* No MySQL migration

==================================================
A. IMAGE INPUT REQUIREMENT
==========================

Update the Academy admin lecture form field:

thumbnailUrl

It must support:

1. رابط صورة

* Admin can paste an http/https image URL.

2. رفع صورة من الجهاز

* Admin can select an image from phone/gallery/device.

Supported file types:

* jpg
* jpeg
* png
* webp

Max size:

* 2 MB

When uploaded:

* Convert the file to base64 data URL on the frontend.
* Store it in the existing thumbnailUrl field.

Do not add external storage.
Do not add Firebase Storage.
Do not add S3.
Do not store videos in DB.

==================================================
C. BACKEND VALIDATION
=====================

Backend must accept both:

* http/https image URL
* base64 data image URL

For base64:
Validate:

* starts with data:image/
* mime is jpeg/png/webp
* decoded size <= 2 MB

For URL:
Validate:

* http or https only
* reject javascript:, file:, blob:, data:text/html, etc.

If an existing shared validator exists, reuse it:
artifacts/api-server/src/lib/imageValidation.ts
or similar.

Apply validation to:

* POST /api/admin/academy/lectures
* PATCH /api/admin/academy/lectures/:id

If thumbnailUrl is empty/null:
Allow it.

==================================================
D. FRONTEND DISPLAY
===================

Academy pages must display thumbnailUrl whether it is:

* http/https URL
* base64 data:image

Affected pages:

* /academy
* /academy/announcements
* /academy/lectures
* /admin/academy list/cards

If image is missing or broken:
Show fallback icon/card background.
Do not show broken image icon.
